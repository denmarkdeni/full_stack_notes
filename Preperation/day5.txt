üåê Introduction to Databases

A database is a structured collection of data 
that can be easily accessed, managed, and updated by applications.

Why Applications Need Databases

Every app needs a place to store and recall data.

üåê Types of Databases

1. Relational Databases (SQL)

Data is stored in tables (rows & columns).

Each table has relationships with others.

Uses SQL (Structured Query Language) for communication.

Examples: MySQL, PostgreSQL, SQLite.

| ID | Name  | Marks |
| -- | ----- | ----- |
| 1  | Arjun | 85    |
| 2  | Maria | 90    |

2. NoSQL Databases

Stores data in a flexible format (not tables).

Best for apps needing speed, scalability, and large unstructured data (like social media).

Examples: MongoDB, Firebase, Cassandra.

{
  "name": "Arjun",
  "marks": 85
}

üåê SQL vs NoSQL - Comparison

| Feature     | SQL (Relational)        | NoSQL (Non-Relational) |
| ----------- | ----------------------- | ---------------------- |
| Data Format | Tables (Rows & Columns) | Documents or Key-Value |
| Structure   | Fixed schema            | Flexible schema        |
| Examples    | MySQL, PostgreSQL       | MongoDB, Firebase      |
| Best For    | Structured data         | Dynamic or large data  |
| Language    | SQL                     | JSON-style queries     |
