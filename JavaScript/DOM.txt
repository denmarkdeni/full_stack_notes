ğŸ“Œ What is BOM?

ğŸ‘‰ BOM = Browser Object Model

ğŸ‘‰ It represents everything the browser provides 
   apart from the HTML document.

ğŸ‘‰ While DOM deals with the page content, 
   BOM deals with the browser window itself.

ğŸ”¹ Main Parts of BOM

ğŸ‘‰ window object

    The global object for everything in the browser.

    All JS running in the browser is inside window.

    Example:

        console.log(window.innerWidth);  // width of the browser window
        console.log(window.innerHeight); // height of the browser window

ğŸ‘‰ navigator object

    Info about the browser & device.

    Example:

        console.log(navigator.userAgent);  // browser details
        console.log(navigator.language);   // browser language

ğŸ‘‰ location object

    Info about the current URL.

    Example:

        console.log(location.href);  // full URL
        location.reload();          // reload the page

ğŸ‘‰ history object

    Controls browser history (back/forward).

    Example:

        history.back();   // go to previous page
        history.forward(); // go to next page

ğŸ‘‰ Dialog Boxes

    alert("Hello!")
    prompt("Enter your name")
    confirm("Are you sure?")

ğŸ”¹ Example Code

  <script>

ğŸ‘‰ BOM is everything the browser gives us to interact

ğŸ”¹ 3. textContent

  ğŸ‘‰ Gets/sets all text, even hidden.

  ğŸ‘‰ Ignores CSS.

ğŸ§© Simple Rule

ğŸ”¹ classList in JavaScript

  ğŸ‘‰ Every DOM element has a classList property, 
  ğŸ‘‰ A special object to manipulate CSS classes.
  ğŸ‘‰ Instead of rewriting the class="" manually, 
    we can add/remove/toggle classes easily.

  âœ… 1. classList.add(className)

    ğŸ‘‰ Adds one or more classes to the element.

    </script>

ğŸ¯ add() â†’ Apply style.

ğŸª Event Bubbling

    Event bubbling is a mechanism in the DOM 
    where an event starts at the target element 
    and then bubbles up (propagates) through its ancestors 
    all the way to the root (document).

  Example:

    <div id="parent">
        <button id="child">Click me</button>
    </div>

    document.getElementById('parent').addEventListener('click', () => {
        console.log('Parent clicked');
    });

    document.getElementById('child').addEventListener('click', () => {
        console.log('Child clicked');
    });

    The click event fires on the button (#child) 
    and bubbles up to the parent (#parent).

ğŸª Event Delegation

    Event delegation is a technique 
    where you attach a single event listener 
    to a common ancestor (often a parent element) 
    instead of multiple child elements. 
    You then use the event.target to determine 
    which child was actually clicked.

    âœ… Benefits:

        Fewer event listeners = better performance

        Easier to handle dynamic elements (added later)

  Example:

    <ul id="menu">
        <li>Home</li>
        <li>About</li>  
        <li>Contact</li>
    </ul>

    document.getElementById('menu').addEventListener('click',(event)=>{
        if (event.target.tagName === 'LI') {
            console.log('You clicked on:', event.target.textContent);
        }
    });

    Even if you later add more <li> elements dynamically, 
    the event listener will still work 
    because it is attached to the parent <ul>.
