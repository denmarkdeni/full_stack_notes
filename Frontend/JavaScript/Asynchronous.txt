.finally() always runs (cleanup, loading spinner stop, etc.).


ðŸ”¹ What Is a Callback?

ðŸ‘‰ A callback is just a function passed as an argument to another 
function, to be executed later.

ðŸ‘‰ JavaScript uses callbacks to handle asynchronous operations 
â€” tasks that take time (like API calls, timers, or file reading).


ðŸ”¹ Example 1: Simple Callback

  console.log("Hello, " + name);

  let name = "Denny";

Async and Await 

ðŸ”¹ What is async?

If you put async before a function, it automatically returns a Promise.

async function sayHello() {
  return "Hello Denny!";
}

sayHello().then(msg => console.log(msg));

ðŸ”¹ What is await?

await tells JavaScript to wait for a Promise to finish before continuing.
It can only be used inside an async function.

function fetchData() {
  return new Promise((resolve) => {
    setTimeout(() => resolve("Data fetched successfully!"), 2000);
  });
}

async function getData() {
  console.log("Start fetching...");
  let result = await fetchData(); // waits here
  console.log(result);
  console.log("All done!");
}

getData();

Output:

Start fetching...
(2-second pause)
Data fetched successfully!
All done!


| Concept     | Meaning                                                | Example                                   |
| ------------| ------------------------------------------------------ | ----------------------------------------- |
| Callback    | Function passed into another function to run later     | `setTimeout(() => {...}, 1000)`           |
| Promise     | Object that represents future value (success or error) | `new Promise((resolve, reject) => {...})` |
| async/await | Cleaner way to write asynchronous code using promises  | `let result = await fetchData();`         |
